## Analysis

### Understanding the Problem

The issue is:
- Daily ingestion is stable with controlled small file counts
- **After rebalance**: small file count suddenly increases
- **Does NOT auto-recover** - requires restart
- After restart, works normally until next rebalance

### Key Variable: `waited_for_assignment`

This variable controls the poll timeout strategy:
- `< 15000ms`: use long timeout (500ms) for better batching
- `>= 15000ms`: use short timeout (50ms) to avoid blocking
